
# ساده‌ترین روش ساخت نسخه ویندوزی بدون کار با خط فرمان

نسخه فعلی برنامه «تطبیق» دارای محیط گرافیکی است. فایل EXE نهایی نیز با گزینه `--windowed` ساخته می‌شود؛ بنابراین هنگام استفاده از برنامه هیچ پنجره Command Prompt باز نمی‌شود.

## روش پیشنهادی: ساخت خودکار در GitHub

1. یک مخزن جدید در GitHub ایجاد کنید.
2. همه محتویات پوشه `tatbigh_app` را در ریشه مخزن بارگذاری کنید.
3. دقت کنید پوشه مخفی `.github` نیز همراه فایل‌ها بارگذاری شود.
4. وارد زبانه `Actions` مخزن شوید.
5. گردش‌کار `Build Tatbigh for Windows` را باز کنید.
6. روی `Run workflow` کلیک کنید.
7. بعد از پایان عملیات، در پایین صفحه بخش `Artifacts` ظاهر می‌شود.
8. فایل `Tatbigh-Windows` را دانلود کنید.
9. فایل ZIP را استخراج و روی `Tatbigh.exe` دوبار کلیک کنید.

کاربر نهایی فقط فایل `Tatbigh.exe` را اجرا می‌کند و با پنجره گرافیکی فارسی کار خواهد کرد.

## دادن پروژه به Work ChatGPT

کل فایل `Tatbigh_MVP_GUI_Source.zip` را بارگذاری کنید و درخواست کنید:
- پروژه را بررسی و خطاهای آن را اصلاح کند.
- رابط را با PySide6 حرفه‌ای‌تر و کاملاً راست‌چین کند.
- نسخه Windows x64 را با PyInstaller و حالت windowed بسازد.
- فایل نهایی `Tatbigh.exe` یا یک Setup نصب‌شونده تحویل دهد.
- هیچ بخشی از استفاده روزانه نیازمند خط فرمان نباشد.

توجه: هر محیط ChatGPT الزاماً امکان کامپایل واقعی فایل Windows EXE را ندارد. وجود گردش‌کار GitHub داخل پروژه، راه مطمئن‌تری برای ساخت خودکار EXE است.
